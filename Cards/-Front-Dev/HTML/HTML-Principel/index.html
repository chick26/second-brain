# HTML 最佳实践

编写易于维护与扩展的 HTML 文档。

# 全局

# 以 DOCTYPE 为开头

激活标准模式需要 DOCTYPE。

Bad:

1
2
3

<html>
  ...
</html>

Good:

1
2
3
4

<!DOCTYPE html>
<html>
  ...
</html>

# 不要使用过时的 DOCTYPE

DOCTYPE 不需要再引用 DTD 了，简单点就行。

Bad:

1
2

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN"
  "http://www.w3.org/TR/html4/strict.dtd">

Good:

1

<!DOCTYPE html>

# 不要使用 XML 作为声明

你确定想写 XHTML?

Bad:

1
2

<?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<!DOCTYPE html>

Good:

1

<!DOCTYPE html>

# 不要什么字符都转义

如果你使用 UTF-8 编写 HTML 文档，那么几乎所有字符（包括表情）都可以直接写。

Bad:

1

<p><small>Copyright © 2014 W3C<sup>®</sup></small></p>

Good:

1

<p><small>Copyright © 2014 W3C<sup>®</sup></small></p>

# 使用字符实体引用来转义 &、<、>、" 和 '

为了 HTML 文档不出错，这些字符应当始终被转义。

Bad:

1

<h1>The "&" character</h1>

Good:

1

<h1>The "&" character</h1>

# 使用字符值引用来转义控制或隐藏字符

这些字符很容易被误认为是其它字符，而且规范也不保证这些字符具有人类可读的名称。

Bad:

1

<p>This book can read in 1 hour.</p>

Good:

1

<p>This book can read in 1 hour.</p>

# 在注释内容周围添加空格

某些字符不能紧接在注释开始或结束的位置上。

Bad:

1

<!--This section is non-normative-->

Good:

1

<!-- This section is non-normative -->

# 别忘了关闭标签

我觉得你不一定知道省略标签关闭的规则。

Bad:

1
2
3

<html>
  <body>
    ...

Good:

1
2
3
4
5

<html>
  <body>
    ...
  </body>
</html>

# 不要杂糅空元素的格式

一致性是可读性的关键。

Bad:

1
2

<img alt="HTML Best Practices" src="/img/logo.png">
<hr />

Good:

1
2

<img alt="HTML Best Practices" src="/img/logo.png">
<hr>

# 不要在标签和属性值周围添加空格

不需要理由。

Bad:

1

<h1 >HTML Best Practices</h1>

Good:

1

<h1>HTML Best Practices</h1>

# 不要杂糅大小写

也是为了一致性。

Bad:

1

<a HREF="#general">General</A>

Good:

1

<a href="#general">General</a>

Also Good:

1

<A HREF="#general">General</A>

# 不要杂糅单双引号

同上。

Bad:

1

<img alt="HTML Best Practices" src='/img/logo.jpg'>

Good:

1

<img alt="HTML Best Practices" src="/img/logo.jpg">

# 不要用多个空格间隔属性

奇怪的格式会把人搞晕的。

Bad:

1

<input   >

Good:

1

<input >

# 省略布尔型属性值

这么写更简单，对吧？

Bad:

1

<audio autoplay="autoplay" src="/audio/theme.mp3">

Good:

1

<audio autoplay src="/audio/theme.mp3">

# 省略命名空间

SVG 和 MathML 可以直接在 HTML 文档中使用。

Bad:

1
2
3

<svg xmlns="http://www.w3.org/2000/svg">
  ...
</svg>

Good:

1
2
3

<svg>
  ...
</svg>

# 不要使用 XML 属性

我们是在写 HTML 文档。

Bad:

1

<span lang="ja" xml:lang="ja">...</span>

Good:

1

<span lang="ja">...</span>

# 别把 data-*、Microdata、RDFa Lite 属性和普通属性混在一起

标签串可以变得很复杂。这条简单的规则有助于阅读这样的标签串。

Bad:

1

<img alt="HTML Best Practices" data-height="31" data-width="88" itemprop="image" src="/img/logo.png">

Good:

1

<img alt="HTML Best Practices" src="/img/logo.png" data-width="88" data-height="31" itemprop="image">

# 首选默认隐式 ARIA 语义

有些元素在 HTML 文档中隐含了某种 ARIA 语义，不要特意把它们指出来。

Bad:

1
2
3
4
5

<nav role="navigation">
  ...
</nav>

<hr role="separator">

Good:

1
2
3
4
5

<nav>
  ...
</nav>

<hr>

# 添加 lang 属性

Good:

# 保持 lang 属性值尽可能简短

Bad:

1

<html lang="ja-JP">

Good:

1

# 尽可能避开 data-*

Bad:

1
2

<span data-language="french">chemises</span>
...

Good:

1
2

<span title="French"><span lang="fr-FR">chemises</span></span>
...

# 文档元数据

# 添加 title 元素

title 元素的值会被很多应用使用，而不仅仅是浏览器。

Bad:

1
2
3

<head>
  <meta charset="UTF-8">
</head>

Good:

1
2
3
4

<head>
  <meta charset="UTF-8">
  <title>HTML Best Practices</title>
</head>

# 不要使用 base 元素

绝对路径或 URL 对开发者和用户来说都更安全。

Bad:

1
2
3
4
5
6

<head>
  ...
  <base href="/blog/">
  <link href="hello-world" rel="canonical">
  ...
</head>

Good:

1
2
3
4
5

<head>
  ...
  <link href="/blog/hello-world" rel="canonical">
  ...
</head>

# 指定次要链接资源的 MIME 类型

这提示了应用要怎么处理这项资源。

Bad:

1
2
3

<link href="/pdf" rel="alternate">
<link href="/feed" rel="alternate">
<link href="/css/screen.css" rel="stylesheet">

Good:

1
2
3

<link href="/pdf" rel="alternate" type="application/pdf">
<link href="/feed" rel="alternate" type="application/rss+xml">
<link href="/css/screen.css" rel="stylesheet">

# 别链接到 favicon.ico

几乎所有浏览器都会自动异步获取 /favicon.ico。

Bad:

1

<link href="/favicon.ico" rel="icon" type="image/vnd.microsoft.icon">

Good:

1

<!-- Place `favicon.ico` in the root directory. -->

# 添加 apple-touch-icon

触摸图标的默认请求路径突然改变了。

Bad:

1

<!-- Hey Apple! Please download `/apple-touch-icon.png`! -->

Good:

1

<link href="/apple-touch-icon.png" rel="apple-touch-icon">

# 给备用样式表添加 title 属性

易读的标签有助于人们选择合适的样式表。

Bad:

1
2

<link href="/css/screen.css" rel="stylesheet">
<link href="/css/high-contrast.css" rel="alternate stylesheet">

Good:

1
2

<link href="/css/screen.css" rel="stylesheet">
<link href="/css/high-contrast.css" rel="alternate stylesheet" title="High contrast">

# 使用 link 元素指向 URL

href 属性的值可以被解析为 URL。

Bad:

1
2
3
4

<section itemscope itemtype="http://schema.org/BlogPosting">
  <meta content="https://example.com/blog/hello" itemprop="url">
  ...
</section>

Good:

1
2
3
4

<section itemscope itemtype="http://schema.org/BlogPosting">
  <link href="/blog/hello" itemprop="url">
  ...
</section>

# 指定文档字符编码格式

UTF-8 暂时还不是所有浏览器的默认值。

Bad:

1
2
3

  <title>HTML Best Practices</title>
</head>

1
2
3
4

# 不要使用过时的字符编码格式

1
